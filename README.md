# Uriel: The Ultimate Third Brain

Uriel is a high-performance, asynchronous Rust bot designed to serve as a frictionless bridge between Discord and your local Obsidian vault. It acts as an intelligent, autonomous agent that ingests notes, tracks entities, prevents duplicate concepts, and synthesizes answers using native Retrieval-Augmented Generation (RAG).

The original Python implementation in this repository has been entirely superseded by the Rust workspace in the `uriel/` directory.

## Features

- **Seamless Vault I/O:** Appends data to Obsidian markdown files safely using OS-level file locking, avoiding sync conflicts.
- **Intelligent Routing (Orichalcum + Gemini):** Uses Gemini Flash-Lite to strictly categorize incoming messages (Note, Draft, Event, Disambiguate, Query, Crawl) into structured JSON.
- **Auto-Backlinking:** Runs a background `walkdir` indexer to find known entities (People, Projects) and automatically creates `[[wikilinks]]` around them in your text.
- **Native RAG:** When asked a question, Uriel securely executes `rg` (ripgrep) to query your vault and synthesizes an answer using Gemini.
- **Connection Crawling:** Utilizing the official `obsidian` CLI, Uriel can explore backlinks and outgoing links for specific concepts to help you navigate your knowledge graph from Discord.
- **Daily Digest Automation:** Runs a daily cron job (06:00 AM) to aggregate the last 7 days of logs, identify patterns, and post an objective to a `#digest` channel using Gemini 1.5 Pro.
- **Multi-Tenant Ready:** Designed to be deployed as multiple independent instances via NixOS systemd services.

## 🚀 Setup & Deployment

### 1. Discord Bot Configuration
1. Go to the [Discord Developer Portal](https://discord.com/developers/applications).
2. Create a new Application and add a "Bot".
3. Under the Bot settings, **enable the "Message Content Intent"**.
4. Generate a bot token (you will need this for `DISCORD_TOKEN`).
5. Invite the bot to your server giving it permissions to Read Messages, Send Messages, and Create Public/Private Threads.
6. Create three specific channels in your server for the bot to interact with. Right-click them to copy their IDs.

### 2. Environment Variables
Uriel requires the following environment variables to run. In a multi-tenant deployment, you can provide different values for different instances.

```env
# Discord Settings
DISCORD_TOKEN="your_discord_bot_token"
DROPBOX_CHANNEL_ID="111111111111111111" # Where you drop notes, media, and unstructured thoughts
DISCUSS_CHANNEL_ID="222222222222222222" # Where you ask Uriel questions to trigger RAG/Crawling
DIGEST_CHANNEL_ID="333333333333333333"  # Where Uriel posts the daily 06:00 AM summary

# Obsidian Vault
VAULT_PATH="/absolute/path/to/your/obsidian/vault"

# AI Intelligence
GEMINI_API_KEY="your_google_gemini_api_key"
```

### 3. Running Locally (Development)
If you want to run or test Uriel locally on your machine:

1. Ensure you have Rust installed.
2. Ensure you have `ripgrep` installed (`rg`).
3. If you intend to use the `Crawl` intent, ensure the official `obsidian` CLI is installed and registered.

Navigate to the `uriel` directory, export your variables, and run:
```bash
cd uriel
export DISCORD_TOKEN="..."
export DROPBOX_CHANNEL_ID="..."
# ... export the rest of the env vars ...

cargo run
```

## 🛠️ Bare-Metal NixOS Deployment (Production)

Uriel is designed to run flawlessly in the background on NixOS, surviving reboots and network drops. The repository includes a `uriel.nix` module for easy systemd integration.

### Prerequisites for NixOS Deployment
- Your NixOS system must be managing secrets (e.g., using `sops-nix`).
- Your secret file must contain the environment variables listed above in standard `KEY=VALUE` format.

### Using the Module

1. Import the `uriel.nix` module into your system configuration.
2. Define the service, pointing to your compiled package and your secrets file.

Example `configuration.nix` snippet for a single instance:

```nix
{ config, pkgs, ... }:
{
  imports = [
    ./path/to/uriel.nix
  ];

  services.uriel = {
    enable = true;
    vaultPath = "/home/user/Documents/MyBrain";
    # Point to the environment file generated by SOPS
    environmentFile = config.sops.secrets."uriel_env".path;
    # Pass the compiled Rust package (e.g., built via a flake)
    package = pkgs.uriel;
  };
}
```

### Multi-Tenant Deployment
Because channel IDs and vault paths are decoupled from the code, you can easily adapt the module (or create multiple `systemd.services` in your Nix config) to run entirely independent instances of Uriel.

For example, you could run one `uriel-work` service listening to a Work Discord server and modifying a Work Vault, and a `uriel-personal` service listening to a Personal Discord server and modifying a Personal Vault, entirely concurrently.

### Monitoring
Once deployed via `nixos-rebuild switch`, you can monitor the bot's health and LLM synthesis logs using `journalctl`:

```bash
journalctl -fu uriel
```
